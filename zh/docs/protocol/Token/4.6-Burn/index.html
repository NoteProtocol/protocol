4.6 销毁 Burn​

Burn操作按照下面的数据定义进行：

{
  "p": "n20",
  "op": "burn",
  "tick": "note",
  "amt": "1000"
  ... ...
}
名称(Key)是否必须(Required?)描述(Description)
p必须协议名称，n20，小写
op必须操作名称，transfer，小写
tick必须代币名称，最大16个字节，大小写不敏感
amt必须Bigint，大整数，销毁的数量，必须大于0
...非必须执行transfer操作时需要的其他变量

如果在deploy阶段设置了智能合约，Burn操作必须能通过合约验证。 如果账号中的Token数量多于销毁数量，则多余的Token将被赋予交易的第一个输出的Token地址。否则，不需要指定Token地址。
